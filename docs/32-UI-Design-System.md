# 32 - UI Design System

**Project:** Ravenshaw Moments\
**Version:** 2.0 (Legacy)\
**Status:** 🔒 Frozen\
**Document Type:** UI Design System

------------------------------------------------------------------------

# Purpose

This document defines the official visual language, design tokens,
component standards, accessibility rules, and interaction guidelines for
Ravenshaw Moments.

------------------------------------------------------------------------

# Design Principles

-   Elegant and timeless
-   Mobile-first
-   Accessible by default
-   Consistent across all modules
-   Performance-focused
-   Emotionally connected to Ravenshaw heritage

------------------------------------------------------------------------

# Brand Identity

Theme:

-   Heritage
-   Academic Excellence
-   Community
-   Memories
-   Trust

------------------------------------------------------------------------

# Color Palette

## Primary

-   Maroon
-   Deep Red
-   White

## Secondary

-   Gold
-   Cream
-   Dark Gray

## Functional

-   Success
-   Warning
-   Error
-   Info

All colors should be exposed as design tokens.

------------------------------------------------------------------------

# Typography

Headings: - Bold - Clear hierarchy

Body: - Highly readable

Monospace: - Code and technical content

------------------------------------------------------------------------

# Spacing Scale

Use an 8px spacing system.

Examples:

-   4
-   8
-   16
-   24
-   32
-   48
-   64

------------------------------------------------------------------------

# Border Radius

-   Small
-   Medium
-   Large
-   Full

Use consistent radius across cards, dialogs and inputs.

------------------------------------------------------------------------

# Shadows

Levels:

-   Subtle
-   Medium
-   Elevated

Avoid excessive shadow usage.

------------------------------------------------------------------------

# Icons

Primary icon library:

-   Lucide Icons

Icons should be consistent in size and stroke.

------------------------------------------------------------------------

# Core Components

-   Buttons
-   Inputs
-   Select
-   Dialog
-   Cards
-   Badges
-   Tables
-   Tabs
-   Toasts
-   Avatars
-   Navigation
-   Breadcrumbs
-   Pagination

------------------------------------------------------------------------

# Layout

Responsive breakpoints:

-   Mobile
-   Tablet
-   Laptop
-   Desktop
-   Wide Screen

Use fluid layouts where possible.

------------------------------------------------------------------------

# Navigation

Primary Navigation

-   Home
-   Departments
-   Hostels
-   Organizations
-   Facilities
-   Competitions
-   Notifications
-   More

Secondary navigation should remain contextual.

------------------------------------------------------------------------

# Forms

-   Clear labels
-   Inline validation
-   Accessible errors
-   Required field indicators
-   Logical grouping

------------------------------------------------------------------------

# Motion

Animations should be:

-   Fast
-   Purposeful
-   Non-blocking

Use subtle transitions.

------------------------------------------------------------------------

# Dark Mode

Support:

-   Light
-   Dark
-   System preference

------------------------------------------------------------------------

# Accessibility

-   WCAG-friendly contrast
-   Keyboard navigation
-   Visible focus
-   Semantic HTML
-   Screen reader compatibility

------------------------------------------------------------------------

# Design Tokens

Maintain centralized tokens for:

-   Colors
-   Typography
-   Radius
-   Spacing
-   Shadows
-   Z-index
-   Animation timing
